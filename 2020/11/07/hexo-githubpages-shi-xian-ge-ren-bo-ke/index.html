使用Hexo搭建静态博客

是配合GitHubPages静态页面，简单静态，挂载在GitHub上即可

安装Node.js

Hexo是基于node.js使用的，所有在使用之前要先安装node环境，安装成功后，在控制台使用命令查看node是否安装成功，使用node -v和 npm -v 查看node版本和npm版本。

添加国内镜像

node.js源都在国外，使用过程中可能速度很慢，会安装失败，可以挂载vpn进行使用，或者使用国内镜像来安装。 npm config set registry https://registry.npm.taobao.org 可添加阿里云镜像。

注册GitHub账号

因为是要借用GitHub的仓库功能来实现静态页面的显示，首先要有一个GitHub账号，没有的话就去 GitHub 注册一个账号。

创建GitHub仓库

登录自己的GitHub账号后，如果使用Hexo，则要创建一个名称有格式要求的仓库进行使用

  • 点击新建一个仓库，名称使用 [yournamer].github.io ，初始化一个readme完成仓库的创建

  • 之后进入当前仓库的 Setting 页面，找到 GitHub Pages 切换一个主题，完成后点击链接查看

  • 出现自己仓库的readme页面信息后表示创建成功】

安装Git

为了能够在本地与GitHub更好的进行文件资源的上传和下载，使用Git软件进行操作，下载安装Git后，在控制台使用命令查看是否安装成功， git --version 查看git安装版本号。

git安装完成后，要自报家门，并且要与自己的github仓库进行关联，使用密钥进行认证

安装Hexo

本地node环境准备好后，进行Hexo的安装，安装过程中使用Git命令工具。

  1. 首先，找一个本地存放Hexo博客文件的位置，如 D:\Blog
  2. 进入到目录中，右键选择 Git Bash Here 打开Git命令窗口
  3. 使用命令 npm i hexo-cli -g 安装Heox，安装完成后使用 hexo -v 查看是否成功

初始化本地blog

安装Hexo后，在同目录下初始化博客网站文件，并进行本地运行效果预览。

  1. 使用命令 hexo init 初始化hexo文件夹

    初始化完成后，本地有了hexo的目录，之后的操作都在根目录下的Git命令窗口中

    初始化时可能会报错，要注意是否使用了管理员权限进行操作

  2. 使用命令 npm install 安装必备的组件

  3. 使用命令 hexo g 生成静态网页

  4. 使用 hexo s 打开本地服务器，开启后可在浏览器 localhost:4000/ 查看本地博客效果

  5. 查看完后，使用 ctrl + c 退出服务

连接本地和远程GitHub

确保本地Git已经和远程GitHub仓库链接成功，打开Hexo根目录下的 _config.yml 文件，修改内容，绑定GitHub指定仓库

开放的博客页面

本地和远程指定仓库绑定后，可以将本地文件内容部署到远程，并通过Pages进行网络访问

  • 安装Git部署插件 npm install hexo-deployer-git --save

  • 使用命令 hexo clean hexo g hexo d

    其中hexo d就是进行网站的部署

  • 访问 [githubname.github.io] 查看部署的网站内容

开始写一篇博客吧

写一篇新的博客时，在Hexo博客根目录下，使用 git bash 打开Git命令界面，在界面操作

  • hexo new post "title" 新增一篇名为title的博客，文件格式为 title.md

    本地文章生成在 D:\Blog\source\_posts 目录下，资源文件都会在source文件夹下

  • 可以使用本地markdown编辑器进行博客内容的书写

博客发布

  • 使用markdown操作完文件后，使用 hexo g 生成静态网页
  • 使用 hexo s 可以开启本地服务器，在 localhost:4000/ 预览效果
  • 使用 hexo d 上传到GitHub上，并部署博客网站，此时可以使用 ..github.io 网络地址进行访问

绑定域名

如果觉得GitHub给的域名不够个性，可以自己购买一个域名，然后绑定到GitHubPages中来，之后可以直接使用自己的域名进行博客页面的访问。

  1. 购买域名并进行域名相关的设置

  2. 在GitHub对应仓库中，添加域名，保存

  3. 设置完成后，本地根目录中出现 CNAME 的文件，内容为域名地址，再使用 hexo g hexo d 部署。

使用过程常见问题

① hexo init blog 时报错

解决办法：将当前文件夹下已经下载的文件删除掉，使用管理员权限重新执行命令即可

② hexo d 时报错

使用 ssh -T git@github.com 是没问题的

解决办法： 一般就是git协议的问题，可以将deploy的地址的http协议换成 ssh协议的

③ 使用不蒜子 进行网络访问计数时，本地服务器运行预览时，访问次数数量巨大

是不蒜子计数的问题，实际部署上去就可以了 不蒜子使用教程
