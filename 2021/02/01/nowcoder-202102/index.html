牛客网02月刷题

0201

1. 计算哈夫曼树的带权路径长度

哈夫曼树的带权路径长度即树中叶子结点的带权路径长度和

  • 结点带权路径长度：该结点到根的路径长度*结点权值
  • 哈夫曼树：给定叶子权值和叶子树，可以构造出不同结构的二叉树，其中带权路径长度最小的二叉树称为最优二叉树，哈夫曼树是一种最优二叉树。
  • 哈夫曼树算法：选取最小结点与已有数形成新树，最终得二叉树。

2. 顺序查找、折半/二分查找、索引/分块查找

三种静态查找算法

  • 顺序查找：查找关键与序列中数逐个比价，直到找到或遍历完成。
    • 顺序遍历整个序列
  • 折半/二分查找：折半/二分查找要求序列在有序的前提下，每次通过与中间位置数值比较舍弃一半数据，比较次数少，查找速度快。
    • 每次舍弃一半的数据
  • 分块/索引查找：分块查找是顺序查找和折半查找结合的一种改进的查找方法，分块查找要求块间有序，块内无序，比较适合结点动态变化的情况。
    • 分块查找需要建立一个索引表，表中存放每个块的起始地址和块中最大元素，以满足块间是有序的。

3. 队列

队列作为一种逻辑结构，他的实现方式有顺序表存储和链表存储两种。

顺序表存储：为解决存储空间浪费的问题，因此顺序表设计为循环队列

链表存储：为带有尾指针的单链表，即可高效入队出队，不需要循环链表。

4. 循环队列（头指针/尾指针）

队头指针front，队尾指针rear

  • 队列的长度计算公式为：((rear-front)+MAXSIZE)%MAXSIZE

5. 普通树转化成二叉树

普通树得到的二叉树的右子树一定为空；森林转换成二叉树有右子树

普通树转成二叉树的步骤是：

  1. 加线：在所有兄弟结点之间添加一条线
  2. 抹线：对树的每个结点，只保留其与第一个子结点之间的连线，删除其他子结点的连线
  3. 旋转：以树的根结点为轴心，整棵树顺时针旋转一定角度，得到二叉树

6. 数据结构的特点

数据结构应有的特点是：插入和删除；

搜索操作不一定有，如栈就没有搜索

7. 堆排序

堆排序的时间复杂度是O(NlogN)，初始建堆复杂度O(N)，调整堆复杂度O(logN)；空间复杂度是O(1)。

8. 图的邻接表

图的邻接表适合存放稀疏图；邻接矩阵适合存放稠密图。

9. JVM虚拟机参数

  • Xmx 最大内存
  • Xms 起始内存
  • Xmn 新生代内存
  • Xss 栈大小（即线程创建后分配给线程的内存大小）
  • -XX:MaxPermSize=n 设置持久代大小
  • -XX:NewRatio=n 设置老年代和年轻代的比值；
  • -XX:ServivorRatio=n 年轻代中Eden区和serivor区（其中一份）的比值

10. 算法的特性

  • 有穷性
  • 确切性
  • 输入项：有0个或多个输入
  • 输出项：至少有一个输出
  • 可行性

11. 计算比较次数

数组中有8个int类型的数据，随机给出数组的数据，找出最大和第二大元素一定需要进行几次比较？

如图：将8个数据两两分组比较，需要7次找到最大值，然后找第二大值需要比较途中黄色部分，比较两次，共7+2=9次。

0202

1. 强连通图

强连通图必须从任何一点出发都可以回到原处，因此：

有n个顶点的强连通图最多有n(n-1)条边，最少有n条边。

2. 邻接表

邻接表中存放的是有入度的结点。

3. B、B-、B+、B*树

  • B树：即二叉搜索树
  • B-树：是一种多路搜索树，就是数据库的索引结构B树
  • B+树：是B-树的变体，也是多路搜索树，非叶子节点作为叶子节点的索引，所有的关键字都在叶子节点上，要到叶子节点才会命中。
  • B*树：B+树的变体，在其基础上，所有的非根非叶子结点也增加了链表指针指向兄弟。

4. Java类加载过程

类方法解析发生在类的加载过程中；成员变量的默认值发生在对象初始化时。

5. Java的开发工具（jar）

Jre中的 JVM不是使用Java实现的，Jdk的开发工具包都是Java写的。

0203

1. Java相关命令工具

  • jsp：查看本机Java进程信息
  • jstack：打印线程的栈信息，制作线程dump文件
  • jmap：打印内存映射，制作堆dump文件
  • jstat：性能监控工具
  • jhat：内存分析工具
  • jconsole：简易的可视化控制台
  • jvisualvm：功能强大的控制台

2. Java匿名内部类

匿名内部类必须继承一个集类，或实现一个接口；

匿名内部类由于没有类名，因此不可以定义构造器；

匿名内部类可以作为形参；

匿名内部类中不能含有静态成员变量和静态方法

3. 循环队列满的判断条件

队头指针为front，队尾指针是rear；

循环队列为空的条件：front=rear;

循环队列为满的条件：(rear+1) % n = front;

因为是循环队列，满时队尾在队头的上一位置。

4. 连通图

有n个点的连通图的边数大于等于n-1；树是一种无向连通图

5. 基于比较的排序算法

  • 基于比较排序的算法有：
    • 直接插入排序
    • 冒泡排序
    • 简单选择排序
    • 希尔排序
    • 快速排序
    • 堆排序
    • 归并排序
  • 基数排序、桶排序都是分配式排序

0204

1. ThreadLocal

ThreadLocal设计的初衷：提供线程内部的局部变量，在本线程内随时随地可取，隔离其他线程。

2. 进制转换

其他进制转换成十进制，和二进制类似，就是a*x^2 + b*x^1 + c*x^0

3. 堆排序算法

堆排序算法属于选择排序，每次都会将根堆（最大值）与尾部值交换，重新调整堆，因此每一轮会确定最大值的位置

4. 希尔排序

希尔排序属于插入类排序，是将整个无序序列分割成若干个小的子序列分布进行插入排序。

5. 图的拓扑序列

有向无环图才有拓扑序列

0205

1. HTTP获取参数方法

  • Cookie[] getCookies()：返回数组，包含客户端发送请求的所有Cookie对象
  • Object getAttribute(String name)：以对象形式返回属性值，如果没有则返回null
  • String getHeader(String name)：以字符串形式返回指定请求头的值，Cookie也是一种请求头。
  • String getParameter(String name)：以字符串形式返回请求参数的值，如果参数不存在则返回null。

2. 邻接表和邻接矩阵（表示图）

  • 邻接表：使用数组+链表的方式存储图，顶点存储在数组中，链表中存放当前点的相邻节点。（邻接表中存放的是有入度的顶点）
  • 邻接矩阵：使用二维数组（矩阵）链存储图，只与图中结点数量有关，值代表两点之间是否有边。

3. 二叉查找树

二叉查找树越复杂（结点稠密），查找效率越高。

4. 关键字比较次数与初始序列无关的排序算法

比较次数与初始序列的关系，即算法的最好和最坏时间复杂度是否一样。

  • 选择排序的最好和最坏时间复杂度都是n^2，即与初始序列无关；
  • 堆排序属于选择排序，也与初始序列无关；
  • 另外，选择排序是不稳定的排序，堆排序也一样。

0206

1. 拓扑排序

有向图才有拓扑排序，拓扑排序是每个结点的所有前驱节点都排在该结点的前边的序列。

2. 关键路径/最短路径/最小生成树

  • 生成树和最小生成树：总的费用最低
  • 拓扑排序：解决工程能否顺利进行（即一个结点要开工必须有前驱结点结束）
  • 关键路径：解决工程完成需要的最短时间问题
  • 最短路径：计算一个结点到其他结点的最短路径，Dijkstra算法

3. 运算符优先级

!、+、-、~、++、– ：运算顺序是从右向左依次运算，即++/–比+/-优先级高

4. 循环队列

循环队列的计算问题：队头指针front、队尾指针rear

  • 队空：front=rear
  • 队满： (rear+1) % maxSize = front
  • 进队：front = (front+1) % maxSize
  • 出队：rear = (rear+1) % maxSize
  • 队列中元素个数：m = (rear-front+maxSize) % maxSize

5. 位示图法

位示图法是利用二进制的一位来表示磁盘中的一个盘块的使用情况

当其值为0时表示对应的盘空闲；

值为1时表示对应盘的已经分配。

0207

1. 字符串的朴素模式匹配算法

串的朴素模式匹配算法也成为BF(Brute-Force)算法，基本思想是：

  • 从主串的第一个字符开始与子串的第一个字符进行比较，
    • 如果相等，则继续比较后续字符；
    • 如不相等，则主串第二个字符与子串第一个字符重新比较；
    • 以此类推，直到子串匹配完成功或主串匹配完失败
  • 最好情况，一次匹配成功
  • 最坏情况：匹配失败，一共匹配 n-m+1 次；主串长n，子串长m

为了解决这种匹配过程中的重复比较问题，引入了**KMP算法**

2. \123

\123表示转义字符，表示八进制中的83，即字符S

3. 数据库表删除语句

  • drop table book：删除整个表
  • truncate table book：删除表中的数据，删除速度比delete更快，无法撤回
  • delete from book：删除表中的数据，可以回退

0208

1. 线索二叉树

定义：一个二叉树通过如下的方法穿起来：

所有原本为空的右孩子指针改为指向该节点在中序序列的后继节点；

所有原本为空的左孩子指针改为指向该节点在中序序列的前驱节点。

即按照中序序列，节点的左指针是前一个，右指针是后一个。

2. 堆排序

堆排序初始建堆的时间复杂度为O(N)，每调整一次复杂度为O(logN)

0209

1. 图的邻接表遍历

图的邻接表存放的时顶点和订单的入度，遍历时是为广度优先遍历，需要借助队列实现。要将每个订单都放入到队列中，此时遍历了所有顶点，针对每个顶点的入度访问到所有的边，因此复杂度为O(n+e)

2. KMP算法

KMP是线性算法，处理主字符串和匹配字符串的复杂度都是O(n)，所有总的复杂度是O(m+n)

3. 数据结构的逻辑结构

逻辑结构即数据元素间抽象化的相互关系，有：

  • 线性结构：线性表
  • 非线性结构：集合、树、图

4. 哈希表的装载因子

装载因子 = 哈希表记录数 / 哈希表长度

0210

1. 矩阵的压缩存储方式

对称矩阵压缩存储时可以只存储上三角形或者下三角形。（一半多）

2. 随机/顺序/索引/散列存取

  • 随机存取–数组
  • 顺序存取–链表
  • 索引/散列存取–哈希表

0211

1. 堆排序的插入法（上浮）和下沉法

  • HeapInsert，插入法，事先不知道有多少个元素，要不断的添加元素来不断调整堆，建堆的时间复杂度为O(NlogN)
  • Heapify，下沉法，知道初始堆长度，从最后一个非叶子节点到根结点进行堆化的调整，不断交换，建堆时间复杂度为O(N)

2. 折半查找判定树

折半查找判定树实际是一棵二叉排序树，其中序序列是一个有序序列

查找时选取比较元素时取整方向要一致。

连通图就只有一个即本身这一个连通分量

串中任意个连续的字符组成的子序列称为该串的子串。

2. 学习英语单词

  • logarithmic：对数的
  • linear：线性的
  • quadratic：二次方的
  • exponential：指数的

0221

1. AOE网

在带权有向图中若以顶点表示时间，有向边表示活动，边上的权值表示该活动持续的时间，这样的图简称为AOE图。（有向无环图）

  • 强连通图：在有向图G中，任意两个顶点都存在路径。
  • 稠密图：指图G的边数接近定点数的平方。
  • 完全图：指一个图的每一对顶点都有边相连。

2. 平衡二叉树

平衡二叉树即AVL树，是二叉搜索树（排序树）的一个升级版，堆不是二叉搜索树。
